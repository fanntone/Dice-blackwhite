# 紅黑大小單雙 · 骰子對決 🎲

2～3 人連線的骰子派對遊戲，純前端、可直接部署到 **GitHub Pages**。

- **連線**：PeerJS（建立／加入房間，免後端、免金鑰）
- **封面**：three.js 把三人合照做成**清晰風格化海報**（調色＋暈影＋模糊沉浸背景＋漂浮 3D 骰子＋視差）
- **同步倒數**：每回合 20 秒倒數，全房同步；逾時自動隨機喊牌換下一位（避免有人卡住）
- **斷線重連**：斷線或重整頁面可自動回到原本牌局、保留座位與骰子
- **離線通知 / 自動換房主**：有人離線會通知其他人（繼續或離開）；**房主離線會自動指派新房主**讓牌局繼續
- **3D 擲骰**：全視窗 three.js 3D 骰子，從外側投入、翻滾、互相碰撞、撞邊框數次後收攏成一排
- **連線強化**：內建 STUN + 免費 TURN 中繼與加入重試，改善「有些人連不到房間」的 NAT 穿透問題

> 只需要 `index.html` + `cover.jpg` 兩個檔即可部署。

---

## 部署到 GitHub Pages

1. 建一個 repo，把 **`index.html`** 和 **`cover.jpg`** 推上去。
2. repo → **Settings → Pages** → Source 選 `main` 分支 `/ (root)` → Save。
3. 幾分鐘後即可用 `https://<帳號>.github.io/<repo>/` 開始玩。

分享：房間頁的「分享邀請」會帶 `?room=代碼` 連結，對方點開自動帶入代碼。

## 本機測試

```bash
python -m http.server 8123     # 在此資料夾啟動
# 瀏覽器開 http://127.0.0.1:8123/
```

測試多人：用**兩支手機**（或兩台電腦／兩個不同瀏覽器）各自「建立／加入房間」。
※ 同一台電腦的兩個分頁會共用瀏覽器身分，請用「無痕視窗」或不同瀏覽器分開測試。

## 連不到房間怎麼辦？

連線走 WebRTC P2P，少數限制較嚴的網路（部分公司／校園／行動網路的對稱式 NAT）需要 **TURN 中繼**才連得上。
程式已內建免費的 STUN/TURN，但免費 TURN 不保證永遠可用。若仍常有人連不到，建議自備 TURN：
改 `index.html` 內的 `ICE_SERVERS`，填入你的 TURN（例如 [metered.ca](https://www.metered.ca/tools/openrelay/) 免費方案）：

```js
const ICE_SERVERS = [
  { urls:'stun:stun.l.google.com:19302' },
  { urls:'turn:你的TURN網域:443', username:'你的帳號', credential:'你的密碼' },
];
```

另外，**房主把手機鎖屏／切到背景**也會讓房間暫時消失（瀏覽器暫停分頁）；請房主保持頁面在前景。

## 封面照片（cover.jpg）

封面會載入同資料夾的 **`cover.jpg`**（三人坐沙發合照），由 three.js 做成風格化海報，並自動裁掉原圖底部字幕／警語。
要換照片就替換 `cover.jpg` 即可；想微調裁切範圍改 `index.html` 內 `Cover` 的 `CROP_MIN / CROP_MAX`。
（沒有 `cover.jpg` 時封面會留白，但遊戲仍可正常進行。）

## 遊戲規則

骰子顏色：**紅 = 1、4**；**黑 = 2、3、5、6**（畫面上紅點即代表「紅」）。

1. 起始玩家看完自己骰子後，從六種喊法擇一：
   紅的拿掉 / 黑的拿掉 / 大的拿掉(4 5 6) / 小的拿掉(1 2 3) / 單數拿掉(1 3 5) / 雙數拿掉(2 4 6)
2. 喊完後全部玩家開盅，依指示拿掉符合的骰子（**含喊者本人**）。
3. 剩餘骰子重新骰動，換下一位玩家喊牌。
4. 直到有人骰子被拿光 → 該玩家為**輸家**。
5. **開局第一次喊牌**就被拿光 → **一擊必殺**，懲罰兩倍。
6. 同時 2 家以上輸掉，依順序**最後一家**為下一場開局玩家。
